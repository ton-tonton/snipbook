# ยกระดับสมองด้วยวิธีคิดนอกกรอบ

**ชื่อต้นฉบับ (ญี่ปุ่น):** 地頭力を鍛える (Jitama Ryoku wo Kitaeru)
**ผู้เขียน:** คิมุระ นาโอโยชิ (Kimura Naoyoshi)
**URL:** https://youtu.be/0qNYqazaehk
**ภาษาในวิดีโอ:** ภาษาไทย


## 📝 สรุปโดยรวม

หนังสือเล่มนี้เขียนโดย **คิมุระ นาโอโยชิ** นักธุรกิจและที่ปรึกษาชาวญี่ปุ่น ที่เคยทำงานที่ **Boston Consulting Group (BCG)** หนึ่งในบริษัทที่ปรึกษาระดับโลก ออกวางจำหน่ายในญี่ปุ่นเมื่อปี **2007** และกลายเป็น bestseller ทันที ขายไปกว่า **1 ล้านเล่ม**

แกนหลักของหนังสือคือ: **ความฉลาดที่แท้จริงไม่ได้อยู่ที่ความรู้ที่ท่องจำ แต่อยู่ที่ความสามารถในการคิด** — โดยเฉพาะการคิดแก้ปัญหาในสถานการณ์ที่ไม่มีข้อมูลครบถ้วน ไม่มีสูตรสำเร็จ และไม่เคยเจอปัญหานั้นมาก่อน


## 📚 แนวคิดหลัก: เจตามะ (地頭力 / Jitama)

**เจตามะ (Jitama)** หรือ "พลังสมองพื้นฐาน" คือแนวคิดสำคัญของหนังสือเล่มนี้ หมายถึงความสามารถในการคิดวิเคราะห์และแก้ปัญหาได้ด้วยตัวเอง โดยไม่ต้องพึ่งข้อมูลครบถ้วนหรือสูตรสำเร็จ

คิมุระสังเกตจากการทำงานกับคนเก่งนับร้อยนับพันคนที่ BCG ว่า:
- คนที่ประสบความสำเร็จจริงๆ **ไม่ได้เก่งเพราะท่องจำเก่ง** หรือขยันอ่านหนังสือ
- แต่เก่งเพราะ **คิดต่าง** มีวิธีคิดที่ทำให้แก้ปัญหาได้เร็ว หาทางออกได้หลากหลาย
- ที่สำคัญคือ **ไม่กลัวปัญหาใหม่** ที่ไม่เคยเจอมาก่อน


## 🧠 ปัญหาของการศึกษาแบบเดิม

ตั้งแต่เด็กๆ เราถูกสอนให้คิดในกรอบ:
- มีคำตอบที่ถูกเพียงคำตอบเดียว
- ข้อสอบมีเฉลย โจทย์มีวิธีแก้ที่แน่นอน
- ถ้าตอบนอกกรอบหรือคิดต่างจากคนอื่น — โดนว่า

พอโตขึ้น เราก็ยิ่งติดกรอบ — กรอบของบริษัท กรอบของสังคม กรอบของความคาดหวัง จนลืมไปว่าสมองของเราสามารถคิดได้กว้างกว่า ลึกกว่า และสร้างสรรค์กว่านี้มาก


## 🛠️ วิธีคิดที่ 1: Framework Thinking (การคิดเป็นโครงสร้าง)

**Framework Thinking** คือการมองปัญหาผ่านโครงสร้างหรือกรอบแนวคิด แทนที่จะคิดแบบวนเวียนหรือสุ่มสี่สุ่มห้า

### ตัวอย่างการใช้ Framework

**ปัญหา:** บริษัทขาดทุน
**คนทั่วไป:** งงไม่รู้จะเริ่มจากไหน
**คนที่ใช้ Framework Thinking:**

```
กำไร = รายได้ - ต้นทุน

ถ้าขาดทุน → รายได้น้อยเกินไป หรือ ต้นทุนสูงเกินไป (หรือทั้ง 2 อย่าง)

รายได้ = จำนวนลูกค้า × ราคาเฉลี่ย
ถ้ารายได้น้อย → ลูกค้าน้อยลง หรือ ลูกค้าซื้อถูกลง
```

ปัญหาใหญ่ๆ เริ่มแตกออกเป็นส่วนย่อยๆ ที่สามารถไปหาข้อมูล วิเคราะห์ และแก้ได้จริง

### ประเภทของ Framework ที่คิมุระแนะนำ

1. **MECE Framework** (Mutually Exclusive, Collectively Exhaustive)
   แปลง่ายๆ: แยกออกจากกัน แต่รวมกันครบถ้วน
   ตัวอย่าง: แบ่งลูกค้าเป็น เด็ก / วัยรุ่น / วัยทำงาน / ผู้สูงอายุ — แต่ละกลุ่มไม่ซ้อนกัน แต่รวมกันครอบคลุมทุกคน

2. **Framework แบบกระบวนการ** — มองปัญหาเป็นขั้นตอน
   ตัวอย่าง: ลูกค้า → รู้จักเรา → สนใจ → ตัดสินใจซื้อ → ใช้ → ซื้อซ้ำ
   ถ้ายอดขายน้อย ปัญหาอยู่ตรงไหน? ไม่มีคนรู้จัก? รู้จักแล้วแต่ไม่สนใจ?

3. **Framework แบบเปรียบเทียบ** — เปรียบเทียบสิ่งต่างๆ เพื่อหาความแตกต่าง
   ตัวอย่าง: เราทำอะไรต่างจากคู่แข่ง? สินค้าขายดีทำอะไรต่างจากสินค้าขายไม่ดี?

### วิธีฝึก Framework Thinking

1. เริ่มจาก Framework พื้นฐาน เช่น แบ่งเป็น ภายใน/ภายนอก, ควบคุมได้/ควบคุมไม่ได้, ระยะสั้น/ระยะยาว
2. **วาดแผนภาพ** — อย่าคิดในหัวอย่างเดียว เอากระดาษมาวาดกล่อง วาดลูกศร วาดความเชื่อมโยง
3. **ดูตัวอย่างจากคนอื่น** — อ่าน Case Study ดูว่าคนอื่นแก้ปัญหายังไง แล้วลองนำมาประยุกต์ใช้

> ทักษะที่แท้จริงคือความสามารถในการ **เลือก Framework ที่เหมาะสม** หรือสร้าง Framework ใหม่ขึ้นมาเอง ไม่มี Framework เดียวที่ใช้ได้กับทุกปัญหา


## 🔭 วิธีคิดที่ 2: Hypothesis Thinking (การคิดแบบสมมติฐาน)

แทนที่จะรวบรวมข้อมูลให้ครบถ้วนก่อนแล้วค่อยสรุป **Hypothesis Thinking** สอนให้:

1. **ตั้งสมมติฐานก่อน** (จากข้อมูลที่มีน้อยนิด)
2. หาข้อมูลเพื่อ **พิสูจน์หรือหักล้าง** สมมติฐานนั้น
3. **ปรับสมมติฐาน** ตามข้อมูลใหม่

### ทำไมต้องคิดแบบนี้?

- ในโลกจริง เราแทบไม่เคยมีข้อมูลครบถ้วน 100%
- รอข้อมูลครบก่อน = เสียเวลา, โอกาส, และ momentum
- การมีสมมติฐานทำให้รู้ว่า "ต้องหาข้อมูลอะไร" ไม่ใช่หาทุกอย่างแบบไม่มีทิศทาง

### ตัวอย่าง

**โจทย์:** ยอดขายตก → ทำอะไรดี?

**แบบเก่า:** รวบรวมข้อมูลทุกอย่างก่อน → วิเคราะห์ → สรุป (ช้า ไม่มีทิศทาง)

**แบบ Hypothesis Thinking:**
→ สมมติฐาน: "น่าจะเป็นเพราะคู่แข่งลดราคา"
→ หาข้อมูลเฉพาะที่ตรวจสอบสมมติฐานนี้
→ ถ้าจริง → วางแผน / ถ้าไม่จริง → ปรับสมมติฐานใหม่


## 📐 วิธีคิดที่ 3: Fermi Estimation (การประมาณโดยไม่มีข้อมูล)

**Fermi Estimation** คือการประมาณค่าที่ดูเหมือนเป็นไปไม่ได้ จากข้อมูลที่มีอยู่น้อยมาก

### ตัวอย่างคำถาม Fermi

- "ในกรุงเทพฯ มีร้านขายกาแฟกี่ร้าน?"
- "คุณสามารถขายสินค้านี้ได้กี่ชิ้นต่อปี?"

**วิธีคิด Fermi:**
1. แบ่งปัญหาออกเป็นส่วนย่อยๆ ที่ประมาณได้
2. ประมาณแต่ละส่วนด้วยเหตุผล
3. รวมกันเพื่อได้คำตอบโดยรวม

ตัวอย่าง (ร้านกาแฟในกรุงเทพฯ):
→ ประชากรกรุงเทพฯ ~10 ล้านคน
→ คนดื่มกาแฟ ~50% = 5 ล้านคน
→ แต่ละคนดื่มเฉลี่ย 1 แก้ว/วัน
→ ร้านหนึ่งขายได้ ~100 แก้ว/วัน
→ ร้านกาแฟ ≈ 5,000,000 / 100 = **50,000 ร้าน**

จุดสำคัญ: คำตอบไม่ต้องแม่นยำ 100% แต่ต้องมี **เหตุผลรองรับ** และอยู่ในลำดับขนาดที่ถูกต้อง (order of magnitude)


## 💡 กรณีศึกษา: พนักงานลาออกเยอะ

คิมุระยกตัวอย่างจากโปรเจคจริง ลูกค้าถามว่า "ทำไมพนักงานลาออกเยอะ?"
ทีมของเขาใช้ Framework Thinking แบ่งปัญหาเป็น 3 ช่วง:

1. **ก่อนเข้างาน** — สรรหาคนผิด ความคาดหวังไม่ตรงกัน
2. **ช่วงทำงาน** — ไม่ตรงความถนัด บรรยากาศไม่ดี ไม่มีการเติบโต
3. **ก่อนออก** — ได้รับข้อเสนอดีกว่าจากที่อื่น

แต่ละช่วงสามารถหาข้อมูล วิเคราะห์ และแก้ไขได้ต่างกัน แทนที่จะมองว่า "พนักงานลาออกเยอะ" เป็นปัญหาก้อนเดียวใหญ่ๆ


## ⚠️ ข้อจำกัดของหนังสือ (มุมมองของผู้รีวิว)

ผู้สรุปวิดีโอให้มุมมองที่น่าสนใจและซื่อสัตย์เกี่ยวกับข้อจำกัด:

1. **ไม่เหมาะกับทุกงาน** — งานที่ต้องการความแม่นยำสูง เช่น งานแพทย์ วิศวกรรม กฎหมาย การสรุปก่อนข้อมูลครบอาจอันตราย

2. **หนังสือเขียนจากมุมมอง Consultant** — สภาพแวดล้อม BCG เปิดกว้างต่อการลองผิดลองถูก แต่บางองค์กรถ้าผิดอาจโดนลงโทษหรือไล่ออก

3. **ไม่ได้พูดถึงข้อจำกัดของบุคคล** — บางคนอาจไม่ถนัดการคิดแบบนี้ หรืออยู่ในสภาพแวดล้อมที่ไม่เอื้อ

> "ผมคิดว่าทุกทักษะมีที่ใช้และไม่ใช้ ไม่มีทักษะไหนที่ใช้ได้ทุกที่ทุกเวลา สิ่งที่สำคัญคือต้องรู้ว่าเมื่อไรควรใช้ และเมื่อไรควรระวัง"


## 👤 หนังสือเล่มนี้เหมาะกับใคร?

**เหมาะกับ:**
1. คนที่รู้สึกว่าตัวเองคิดช้า แก้ปัญหาไม่เป็น เจอปัญหาใหม่แล้วไม่รู้จะเริ่มจากไหน
2. คนทำงานที่ต้องแก้ปัญหาบ่อยๆ โดยเฉพาะงานที่เจอปัญหาใหม่ตลอด เช่น Consultant, Startup, งานนวัตกรรม
3. คนที่อยากพัฒนาตัวเอง อยากเรียนรู้สิ่งใหม่ได้เร็วขึ้น

**ไม่เหมาะกับ:**
1. งานที่มีขั้นตอนชัดเจนอยู่แล้ว มีคู่มือ มีสูตร แค่ทำตามก็พอ
2. งานที่ต้องการความแม่นยำสูงมาก ผิดพลาดนิดเดียวเป็นอันตราย
3. คนที่ไม่พร้อมจะฝึก — เพราะหนังสือนี้ไม่ใช่แค่อ่านแล้วจบ ต้องลงมือฝึกจริง
